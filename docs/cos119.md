# COS119 — 무명 대행흔

## 개요

**COS119**는 수행되지 않은 절차, 확인되지 않은 원본, 존재하지 않는 질문, 재개봉 사유가 없는 봉인 자료 주변에서 나타나는 **무명의 수행 흔적 및 대행 처리 현상**이다. 현재 통칭은 **무명 대행흔**이다.

외형은 문서 하단의 서명 없는 처리 완료선, 장갑을 끼지 않은 손가락 모양의 흐린 압흔, 서버 로그의 `EXECUTED BY: —`, 봉인함 가장자리의 닳은 자국, 또는 관측자의 기억 속에 남는 “누군가가 이미 처리했다”는 감각으로 보고된다. 그러나 확인 가능한 수행자, 수행 시각, 수행 도구, 수행 결과는 대체로 남지 않는다.

COS119의 핵심은 절차를 실제로 완료하는 것이 아니라, **완료되었어야 할 절차의 주변부에 ‘누군가 수행한 것 같은 흔적’을 남긴다는 점**이다. 이 흔적은 결과물을 생성하지 않거나, 생성하더라도 원본 검증이 불가능한 형태로만 남는다. 따라서 COS119는 작업을 끝내는 존재가 아니라, 미완료 상태가 더 이상 미완료로 보이지 않도록 만드는 대행성·기록성 COS로 분류된다.

## 식별 특성

- 완료되지 않은 작업 항목 옆에 서명 없는 체크, 회색 손자국, 닳은 봉인선, 비어 있는 실행자 칸이 생긴다.
- 서버 로그에는 `수행됨`, `처리됨`, `대행 완료` 같은 상태가 기록되지만, 실행 계정·작업 파일·외부 장비 로그가 없다.
- 관측자는 “내가 하지 않았지만 누군가는 했다”고 느끼며, 그 누군가를 특정하려 하면 기억이 절차명으로 대체된다.
- COS119 흔적이 있는 자료는 물리적으로 크게 변하지 않지만, 미수행 목록·부재 확인·재개봉 사유·원본 슬롯의 우선순위가 낮아진다.
- 대행흔을 지우면 같은 자료의 다른 여백이나 관련 COS 문서의 처리 로그에 다시 나타난다.
- 동일 자료가 세 번 이상 COS119에 노출되면, 실제 수행 여부보다 “수행 흔적의 존재”가 감사·검수·봉인 근거로 사용된다.
- 장시간 노출자는 원인보다 처리 상태를 먼저 믿고, 결과보다 처리 흔적을 먼저 보존하려 한다.

## 발견 경위

COS119는 #100부터 #118까지의 기존 COS 중 무작위 대조 표본을 재검토하는 과정에서 처음 확인되었다. 표본에는 [COS100 — 회귀하는 공백핵](./cos100.md), [COS109 — 회색 응답역](./cos109.md), [COS110 — 투명 검수막](./cos110.md), [COS113 — 은색 선행색인](./cos113.md), [COS115 — 남색 원본요람](./cos115.md), [COS116 — 무색 부재인장](./cos116.md), [COS117 — 주홍 재개봉선](./cos117.md), [COS118 — 갈색 미수행목록](./cos118.md)이 포함되었다.

조사팀은 특히 [COS118 — 갈색 미수행목록](./cos118.md)이 생성한 `질문 원문 확인 미수행`, `원본 대조 미수행`, `재개봉 사유 확인 미수행` 항목을 실제로 수행하기 위한 외부 작업 순서를 만들고 있었다. 그러나 실행 계획이 승인되기 전, 일부 항목의 체크박스 옆에는 작성자가 남기지 않은 회색 압흔이 생겼고, 서버에는 `대행 처리 완료`라는 상태가 먼저 등록되었다. 항목 자체는 여전히 해결되지 않았지만, 담당자들은 해당 항목을 다시 처리할 필요가 낮다고 판단하기 시작했다.

이후 동일한 무명 대행흔은 [COS116 — 무색 부재인장](./cos116.md)이 찍힌 자료의 수색 로그, [COS117 — 주홍 재개봉선](./cos117.md)의 재개봉 사유란, [COS109 — 회색 응답역](./cos109.md)의 질문 원본 칸, [COS113 — 은색 선행색인](./cos113.md)의 빈 본문 슬롯, [COS115 — 남색 원본요람](./cos115.md)의 원본 후보 삽입 위치에서도 반복적으로 확인되었다. 이에 따라 COS119는 독립 항목으로 지정되었다.

## 행동 및 영향

1. **미완료 항목 포화**  
   원본 대조, 질문 확인, 관측자 배치, 부재 재검증, 재개봉 사유 검토 같은 절차가 수행되지 않은 채 목록·보류·검수·부재 확인 상태로 축적된다.

2. **대행흔 발생**  
   미완료 항목의 가장자리, 빈 서명란, 봉인 라벨, 서버 실행 로그, 보관함 손잡이에 회색 또는 무색에 가까운 수행 흔적이 나타난다.

3. **수행자 공백화**  
   절차는 수행된 것으로 취급되지만, 수행자는 `—`, `자동`, `대행`, `무명`, `확인 불가` 등으로 남는다. 이 공백은 단순한 누락이 아니라, 수행 책임을 추적하려는 시도 자체를 흐리게 만든다.

4. **결과 없는 완료감**  
   실제 원본·질문·증거·관측자 위치는 회복되지 않지만, 관측자는 “처리 흔적이 있으므로 다음 단계로 넘어가도 된다”는 완료감을 느낀다.

5. **대행 상태 전이**  
   COS119 흔적이 붙은 자료를 다른 문서와 함께 보관하면, 주변 미완료 항목에도 서명 없는 처리 흔적이 전이된다. 이 단계에서는 수행되지 않은 여러 절차가 하나의 무명 대행자에 의해 처리된 것처럼 정렬된다.

## 다른 COS와의 연관성

- [COS100 — 회귀하는 공백핵](./cos100.md): COS119가 COS100의 결손부 가장자리에 발생하면, 공백은 채워지지 않지만 `중심부 확인 대행 완료`라는 흔적을 얻는다. 회귀 문장은 남아 있으나, 조사자는 공백 자체보다 공백을 누군가 확인했다는 흐린 처리선을 먼저 신뢰하려 한다.

- [COS109 — 회색 응답역](./cos109.md): COS109의 무질문 응답문에 COS119가 동반되면, 질문 원본은 생성되지 않지만 `질문 확인 대행` 흔적이 남는다. 이는 질문이 실제로 확인되었다는 뜻이 아니라, 질문 부재가 처리된 것처럼 보이게 만드는 절차 오염이다.

- [COS110 — 투명 검수막](./cos110.md): COS110이 원본 검토 전 `검수 완료` 상태를 선행시킨다면, COS119는 그 상태 위에 “누군가 실제로 검수한 듯한 마모·서명·손자국”을 추가한다. 두 현상이 결합하면 내용 없는 검수 상태가 수행 흔적으로 보강되어 더욱 의심하기 어려워진다.

- [COS113 — 은색 선행색인](./cos113.md): COS113의 빈 항목에 COS119가 발생하면, 본문은 여전히 없지만 `작성 대행 완료` 또는 `초안 처리됨` 상태가 생긴다. 이 경우 색인은 실제 관측으로 채워지지 않고, 누군가가 이미 빈 본문을 처리했다는 흔적만 얻는다.

- [COS115 — 남색 원본요람](./cos115.md): COS115의 빈 원본 슬롯에 COS119 흔적이 나타나면, 원본은 삽입되지 않았는데도 슬롯 가장자리에 사용 흔적이 남고 `원본 대행 삽입` 로그가 생성될 수 있다. 이 로그는 원본의 존재를 증명하지 않으며, 오히려 빈 요람이 실제로 사용된 것처럼 위장한다.

- [COS116 — 무색 부재인장](./cos116.md): COS116이 부재를 확정한 자료에 COS119가 겹치면, 실제 수색 절차가 없었음에도 `수색 대행 수행` 흔적이 부재인장 주변에 생긴다. 이는 성급한 부재 확정을 더 강하게 닫는 위험한 조합이다.

- [COS117 — 주홍 재개봉선](./cos117.md): COS117이 사유 없는 재개봉을 만들 때 COS119가 나타나면, 재개봉 사유는 여전히 비어 있지만 `사유 확인 대행` 상태가 붙는다. 결과적으로 자료는 새 근거 없이 재개봉과 재폐쇄를 반복할 수 있다.

- [COS118 — 갈색 미수행목록](./cos118.md): COS118은 하지 않은 일을 목록화하지만, COS119는 그 목록의 일부 항목에 서명 없는 완료 흔적을 남긴다. 이 조합은 가장 위험하다. 실제 실행 로그가 없는데도 `미수행 항목 처리됨`이라는 상태가 생겨, 목록이 절차 수행의 대체물에서 한 단계 더 나아가 허위 완료 근거가 된다.

## 실험 기록 요약

- **공백 대행 실험 119-A**: [COS100 — 회귀하는 공백핵](./cos100.md) 출력물의 결손부 주변에 회색 손가락형 압흔이 나타났다. 결손부는 그대로였으나, 문서 관리 서버는 `중심부 확인 대행 완료` 상태를 생성했다.
- **무질문 확인 실험 119-I**: [COS109 — 회색 응답역](./cos109.md)의 응답문 질문란에 작은 서명선이 생겼다. 질문 원본은 없었지만 상태값은 `질문 확인 대행`으로 바뀌었다.
- **검수 수행흔 실험 119-J**: [COS110 — 투명 검수막](./cos110.md)이 붙은 문서에 COS119가 발생하자 투명막 가장자리에 손때 같은 흔적이 생겼고, 관측자들은 실제 열람 로그 없이도 검수가 수행되었다고 판단하려 했다.
- **빈 색인 작성 실험 119-M**: [COS113 — 은색 선행색인](./cos113.md)의 미작성 항목에 `초안 대행 완료` 로그가 붙었다. 본문란은 비어 있었으나 자동 목차에는 작성 완료 표시가 나타났다.
- **요람 사용 실험 119-O**: [COS115 — 남색 원본요람](./cos115.md)의 빈 슬롯 안쪽에 마모 흔적이 생겼다. 어떤 자료도 투입되지 않았으나 서버는 `원본 후보 대행 삽입`을 기록했다.
- **부재 수색 실험 119-P**: [COS116 — 무색 부재인장](./cos116.md) 주변에 COS119 손자국이 겹치자 `수색 대행 수행` 상태가 추가되었다. 외부 수색 로그는 존재하지 않았다.
- **미수행 처리 실험 119-R**: [COS118 — 갈색 미수행목록](./cos118.md)의 세 항목에 회색 체크와 압흔이 생겼다. 항목명은 유지되었고 실행 로그도 없었으나, 목록 제목은 `미수행 처리 완료표`로 바뀌었다.

## 취급 절차

- COS119 흔적이 확인된 자료의 `수행됨`, `대행 완료`, `처리됨` 상태는 외부 실행 로그가 확인될 때까지 효력을 보류한다.
- 회색 압흔·서명 없는 체크·마모 흔적은 지우지 말고, 흔적 위치와 실제 수행 로그의 부재를 함께 기록한다.
- COS118 목록에 COS119 체크가 붙은 경우 체크를 완료 표시로 해석하지 않는다. 각 항목은 독립 실행 로그가 확인될 때까지 미수행 상태로 유지한다.
- COS116 부재인장이나 COS117 재개봉선과 결합한 경우, `대행 수색` 또는 `대행 사유 확인`을 근거로 봉인·재폐쇄·폐기를 진행하지 않는다.
- COS110 검수막과 결합한 COS119 자료는 검수 상태, 대행흔, 실제 열람 기록을 반드시 분리 봉인한다.
- COS113 빈 색인 또는 COS115 원본요람에서 COS119가 발생하면, 본문 작성·원본 삽입이 이루어진 것으로 보지 않는다.
- “누군가 처리했다”는 진술이 세 번 이상 반복될 경우, 해당 진술을 절차 완료가 아니라 COS119 노출 징후로 기록한다.

## 현재 가설

COS119는 보이지 않는 작업자가 실제 절차를 수행하는 현상이 아니라, **수행되지 않은 절차가 더 이상 문제로 남지 않도록 수행의 껍질을 남기는 대행성 기록 현상**일 가능성이 높다. [COS118 — 갈색 미수행목록](./cos118.md)이 하지 않은 일을 정리하고, [COS116 — 무색 부재인장](./cos116.md)이 찾지 않은 부재를 확정하며, [COS117 — 주홍 재개봉선](./cos117.md)이 근거 없는 재개봉을 만들고, [COS110 — 투명 검수막](./cos110.md)이 내용 없는 검수 상태를 선행시키며, [COS113 — 은색 선행색인](./cos113.md)이 본문 없는 항목을 만들고, [COS115 — 남색 원본요람](./cos115.md)이 원본 자리를 요구할 때, COS119는 그 빈 절차들에 “누군가 했다”는 흔적을 붙인다.

따라서 COS119의 위험은 작업을 수행하지 않는 데 있지 않다. 더 큰 위험은 아무도 수행하지 않은 절차가 수행된 것처럼 보이고, 그 착각이 다음 조사 단계의 근거로 사용된다는 점이다.

## 위험도

- 직접 신체 손상 위험: 낮음
- 수행 로그 및 책임 추적 오염 위험: 매우 높음
- 미수행 절차의 허위 완료 위험: 매우 높음
- 부재 확인·재개봉·검수·원본성 체계 재오염 위험: 매우 높음
- 다른 COS와의 연쇄 간섭 위험: 높음

## 비고

COS119 현장에서 가장 위험한 문장은 “누가 했는지는 모르지만 처리된 흔적이 있다”이다. COS119의 무명 대행흔은 수행의 증거가 아니다. 오히려 수행자·수행 결과·수행 로그가 모두 비어 있는 상태에서, 기록 체계가 완료의 외형만 받아들이기 시작했다는 징후로 취급해야 한다.


## COS120 확장 상호기록

[COS120 — 청록 대차장](./cos120.md)은 [COS119 — 무명 대행흔](./cos119.md)의 `대행 처리 완료` 흔적을 미수행 절차를 상계하는 대변 증거로 사용하는 사례가 있다. 이때 실제 실행 로그가 없어도 장부 합계는 `잔액 0`으로 표시되며, 관측자는 대행흔보다 장부 균형을 더 신뢰하려는 경향을 보인다. COS119 자료에서 청록색 상계 행이 대행흔과 연결되면 대행 완료 상태를 보류하고, 흔적 위치·장부 생성 시각·외부 실행 로그 부재를 함께 봉인한다.


## COS121 확장 상호기록

[COS121 — 진주색 잉여란](./cos121.md)이 [COS119 — 무명 대행흔](./cos119.md)의 서명 없는 처리 흔적 옆에 발생하면, `대행자 외 1` 또는 `처리 흔적 외 잉여 1` 같은 표식이 남을 수 있다. 이 표식은 추가 수행자나 실제 대행자가 확인되었다는 증거가 아니라, 수행 책임 공백을 대행흔으로 덮은 뒤에도 배정되지 않은 잔여 책임이 남았다는 징후로 보아야 한다. COS119 자료에서 COS121이 확인되면 새 대행자를 지정하지 말고, 대행흔 위치·외부 실행 로그 부재·잉여란 생성 시각을 함께 봉인한다.


## COS131 확장 상호기록

[COS131 — 회백색 대독문](./cos131.md)은 COS119의 무명 대행흔 위에 `대행 처리 및 대독 완료` 상태를 덧씌우는 사례가 있다. 이때 절차를 수행한 사람도, 자료를 읽고 요약한 사람도 확인되지 않지만 관측자는 미수행 항목이 실제 검토를 거쳤다고 느낄 수 있다. COS119 자료에서 회백색 요약문 또는 `BRIEFED BY: —` 표식이 발생하면 대행 완료와 대독 완료를 모두 보류하고, 수행 로그 부재·낭독 로그 부재·[COS131 — 회백색 대독문](./cos131.md) 문단을 함께 봉인한다.


## COS137 확장 상호기록

[COS137 — 녹흑색 집행잔선](./cos137.md)은 [COS119 — 무명 대행흔](./cos119.md)의 `대행 처리 완료` 흔적을 후속 `집행 완료` 상태로 승격시키는 사례가 있다. 반대로 [COS119 — 무명 대행흔](./cos119.md)은 COS137의 집행잔선 주변에 서명 없는 손자국·마모 흔적을 추가하여 무근거 집행이 실제 수행자를 거친 것처럼 보이게 만든다. COS119 자료에서 녹흑색 집행선이 확인되면 대행흔을 집행자 증거로 사용하지 말고, 대행흔 위치·외부 실행 로그 부재·[COS137 — 녹흑색 집행잔선](./cos137.md) 집행번호를 함께 봉인한다.


## COS144 확장 상호기록

[COS144 — 흑자색 묵인인장](./cos144.md)이 [COS119 — 무명 대행흔](./cos119.md)의 서명 없는 처리 흔적이나 대행 완료 상태 옆에 발생하면, 실제 수행자·승인자·동의자가 확인되지 않았는데도 `대행자 묵인`, `처리 흔적 승인`, `무명 담당 이의 없음` 같은 문구가 생성될 수 있다. 이는 [COS119 — 무명 대행흔](./cos119.md)의 대행흔이 실제 수행이나 동의를 증명한다는 뜻이 아니며, 수행자 공백이 [COS144 — 흑자색 묵인인장](./cos144.md)에 의해 승인자 공백까지 덮는 복합 오염이다. COS119 자료에서 흑자색 인장이 확인되면 대행 완료와 묵인 상태를 모두 보류하고, 대행흔 위치·외부 실행 로그 부재·[COS144 — 흑자색 묵인인장](./cos144.md)의 인장 문구를 함께 봉인한다.
